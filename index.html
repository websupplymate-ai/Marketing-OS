How modules connect
Dashboard (frontend shell) → /api/competitor-insights /api/content-planner /api/scriptwriter /api/whatsapp /api/calling /api/meta-ads /api/google-ads /api/scraper → each module's own DB
Competitor file convention: every uploaded workbook for Competitor Insights is parsed as two sheets — p = purchase data, s = sale data. The ingestion layer should validate both sheets exist, join them on SKU/date, and write to the same competitor-db record so margin and pricing views stay in sync.
